# Lab 1 - Okta User Provisioning

## Objective

The objective of this lab was to understand how user accounts are created and managed in Okta, a leading Identity and Access Management (IAM) platform.

## Lab Activities

* Logged into the Okta Admin Console
* Navigated to Directory → People
* Created new user accounts
* Reviewed user profile information
* Verified successful user creation

## IAM Concepts Covered

### Identity

An identity represents a digital user within an organization.

### User Provisioning

Provisioning is the process of creating user accounts and establishing identities in an IAM system.

### User Directory

Okta maintains user information such as:

* First Name
* Last Name
* Username
* Email Address
* Account Status

## Business Use Case

When a new employee joins an organization, the IAM team provisions a user account in the identity platform. This account can later be assigned to groups, applications, and access permissions based on business requirements.

## Skills Demonstrated

* Okta Administration (Basic)
* User Provisioning
* Identity Management
* IAM Fundamentals

## Tools Used

* Okta Identity Cloud
* Web Browser

## Outcome

Successfully created user accounts in Okta and gained hands-on experience with identity provisioning and user management workflows.
