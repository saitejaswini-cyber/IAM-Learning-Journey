# User Management Best Practices

## Overview

User Management Best Practices help organizations maintain secure, efficient, 
and compliant identity administration. 
Following these practices reduces security risks and ensures
users have the appropriate level of access throughout their lifecycle.

---

## 1. Principle of Least Privilege

Grant users only the minimum level of access required to perform their job responsibilities.

### Benefits

- Reduces security risks
- Prevents unauthorized access
- Supports regulatory compliance

---

## 2. Use Strong Password Policies

Implement password policies that require:

- Minimum password length
- Uppercase and lowercase letters
- Numbers
- Special characters
- Regular password changes (if required by policy)

---

## 3. Enable Multi-Factor Authentication (MFA)

Require users to verify their identity using more than one authentication factor.

Examples:

- Authenticator App
- SMS OTP
- Security Key
- Biometric Authentication

---

## 4. Review User Access Regularly

Perform periodic access reviews to ensure users only have access required for their current role.

Examples:

- Quarterly access reviews
- Annual access certification
- Manager approval

---

## 5. Remove Unused Accounts

Disable or delete inactive accounts promptly.

Examples:

- Former employees
- Temporary contractors
- Dormant accounts

---

## 6. Maintain Accurate User Information

Keep user profile information up to date.

Examples:

- Department
- Job Title
- Manager
- Email Address
- Employment Status

---

## 7. Monitor User Activity

Review login activity and account events to identify unusual behaviour.

Examples:

- Failed login attempts
- Suspicious sign-in locations
- Account lockouts

---

## 8. Follow Joiner-Mover-Leaver (JML)

Ensure user access is updated throughout the employee lifecycle.

- Joiner → Provision access
- Mover → Update permissions
- Leaver → Remove access immediately

## Interview Questions

### Why is the Principle of Least Privilege important?

**Answer:**

It ensures users receive only the permissions necessary to perform their job, 
reducing security risks and limiting the impact of compromised accounts.

---

### Why should inactive accounts be removed?

**Answer:**

Inactive accounts can become security risks if they are forgotten or compromised. 
Removing or disabling them reduces the attack surface.

---

## Summary

Applying User Management Best Practices improves security, supports compliance,
simplifies administration, and helps organizations effectively manage user identities throughout their lifecycle.
