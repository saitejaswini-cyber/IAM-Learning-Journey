# User Lifecycle Management

## Overview

User Lifecycle Management (ULM) is the process of managing a
user's identity throughout their entire employment within an organization.
It ensures that users have the appropriate access at every stage of their
lifecycle while maintaining security and compliance.

---

## User Lifecycle Stages

### 1. Joiner

A new employee joins the organization.

Activities:

- Create user account
- Assign username
- Assign email address
- Add user to appropriate groups
- Assign required applications
- Enable Multi-Factor Authentication (MFA)

---

### 2. Mover

An existing employee changes role, department, or location.

Activities:

- Update user profile
- Modify group memberships
- Remove unnecessary access
- Grant new permissions
- Update application assignments

---

### 3. Leaver

An employee leaves the organization.

Activities:

- Disable user account
- Revoke application access
- Remove group memberships
- Archive user data (if required)
- Delete or deactivate the account according to company policy

---

## Why is User Lifecycle Management Important?

- Protects organizational data
- Prevents unauthorized access
- Supports compliance requirements
- Simplifies identity administration
- Automates onboarding and offboarding

## Interview Questions

### What is User Lifecycle Management?

User Lifecycle Management is the process of managing user
identities and access rights from the time a user
joins an organization until they leave. 
It includes user provisioning, role changes, access updates, and account deactivation.

---

### What are the three stages of User Lifecycle Management?

- Joiner
- Mover
- Leaver (JML)

---

## Summary

User Lifecycle Management ensures that users receive the correct access throughout their employment while protecting organizational resources through proper identity and access management practices.
